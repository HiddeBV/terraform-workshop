# Setup Guide - Terraform Workshop Day 4

This guide will help you set up and complete the Terraform Workshop successfully.

## 🛠️ Prerequisites

### Required Tools
- **Terraform** v1.12.2+ ([Download](https://terraform.io/downloads))
- **Azure CLI** ([Install Guide](https://docs.microsoft.com/en-us/cli/azure/install-azure-cli))
- **terraform-docs** v0.20+ ([Install Guide](https://terraform-docs.io/user-guide/installation/))

### Azure Requirements
- Azure subscription with appropriate permissions
- Ability to create resource groups, storage accounts, and other resources

## 🚀 Quick Start

### 1. Authentication
```bash
# Login to Azure
az login

# Set your subscription (if you have multiple)
az account set --subscription "your-subscription-id"

# Verify current subscription
az account show
```

### 2. Clone and Navigate
```bash
git clone <repository-url>
cd terraform-workshop/day4-lab
```

### 3. Sprint-by-Sprint Execution

#### Sprint 1: Remote Backend (⏱ ~30 min)

1. **Navigate to backend module:**
   ```bash
   cd 00-backend
   ```

2. **Bootstrap the backend infrastructure (Recommended):**
   ```bash
   cd bootstrap/
   ./bootstrap.sh
   ```
   Follow the generated migration instructions to import and configure the remote backend.

   **Alternative: Manual setup:**
   ```bash
   cp terraform.tfvars.example terraform.tfvars
   # Edit terraform.tfvars with your values
   terraform init
   terraform plan
   terraform apply
   ```

3. **Save outputs:**
   ```bash
   terraform output > ../backend-outputs.txt
   ```

4. **Verify Definition of Done ✅:**
   - [ ] `terraform init` prints "Successfully configured the backend 'azurerm'"
   - [ ] Tests pass
   - [ ] README generated by terraform-docs
   - [ ] Backend infrastructure is managed by Terraform

#### Sprint 2: Network (⏱ ~45 min)

1. **Navigate to network module:**
   ```bash
   cd ../01-network
   ```

2. **Configure backend:**
   ```bash
   cp backend.hcl.example backend.hcl
   # Update backend.hcl with values from Sprint 1 outputs
   ```

3. **Create terraform.tfvars:**
   ```bash
   cp terraform.tfvars.example terraform.tfvars
   # Edit terraform.tfvars with your values
   ```

4. **Deploy:**
   ```bash
   terraform init -backend-config=backend.hcl
   terraform plan  # Should show 5 resources
   terraform apply
   ```

5. **Verify Definition of Done ✅:**
   - [ ] `terraform plan` shows 5 resources
   - [ ] Tests & docs pass

#### Sprint 3: Key Vault & IAM (⏱ ~45 min)

1. **Navigate to keyvault module:**
   ```bash
   cd ../02-keyvault
   ```

2. **Configure backend and variables similarly to Sprint 2**

3. **Deploy and verify Key Vault is alive and secrets are available**

4. **Verify Definition of Done ✅:**
   - [ ] Key Vault alive, secret available
   - [ ] Tests & docs pass

#### Sprint 4: App Service (⏱ ~70 min)

1. **Navigate to app module:**
   ```bash
   cd ../03-app
   ```

2. **Configure backend and variables**

3. **Deploy:**
   ```bash
   terraform init -backend-config=backend.hcl
   terraform plan  # Should show 3 resources
   terraform apply
   ```

4. **Test deployment:**
   ```bash
   APP_URL=$(terraform output -raw app_service_url)
   curl $APP_URL  # Should return HTTP 200
   ```

5. **Verify Definition of Done ✅:**
   - [ ] `terraform apply` creates/updates resources and deployment succeeds
   - [ ] Browser shows "Hello World – DB OK"
   - [ ] `terraform apply` creates 3 resources
   - [ ] `curl <app-url>` returns HTTP 200
   - [ ] Tests & docs pass

#### Sprint 5: SQL Database & Private Endpoint (⏱ ~50 min)

1. **Navigate to data module:**
   ```bash
   cd ../04-data
   ```

2. **Configure and deploy**

3. **Test private connectivity:**
   ```bash
   nslookup $(terraform output -raw sql_server_fqdn)
   # Should resolve to 10.x address
   ```

4. **Verify Definition of Done ✅:**
   - [ ] Public access to port 1433 blocked
   - [ ] `nslookup <db-fqdn>` resolves to 10.x address
   - [ ] Tests & docs pass

## 🔧 Troubleshooting

### Common Issues

### Backend Configuration Errors
- Ensure you've completed Sprint 1 first
- **For bootstrap approach**: Follow the migration instructions generated by `bootstrap.sh`
- **For manual approach**: Verify backend.hcl has correct values from Sprint 1 outputs
- Check Azure CLI authentication

#### Permission Issues
- Verify Azure subscription permissions
- Ensure service principal has appropriate roles
- Check resource naming conflicts

#### Terraform State Issues
- Use `terraform state list` to see current resources
- Use `terraform refresh` to update state
- Check backend storage account accessibility

### Getting Help

1. Check module-specific README files
2. Review test output in `day4-lab/test/test.out`
3. Verify Azure resource configuration in portal
4. Use `terraform plan` to preview changes
